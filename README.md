# in喵剧本管理工作台

> 面向长剧编剧团队的纯本地剧本管理工具。专攻「文档分析、信息整合与查找」，无需安装，浏览器即开即用。

## 为什么做这个项目

长剧项目（通常 30–50 集）的创作过程中，编剧团队需要处理海量文档与信息。我们也曾尝试寻找合适的工具来提升效率，但市面上的产品往往无法真正匹配需求：

- AI 创作类工具：创意是编剧的核心工作，无法交由 AI 替代。且长剧项目信息量庞大、条件苛刻，使用 AI 创作反而需要反复校验、调整提示词，时间成本更高。
- 自动格式类工具：主要解决排版与格式规范问题，但难以适配团队特定的剧本格式要求。
- 制片向解析软件：面向统筹与制片流程设计，功能重心在预算、通告与资源调度，对编剧团队的日常创作帮助有限。

真正占用大量时间、却又最"可被工具替代"的，是「文档分析、信息归纳与查找」这类低价值重复劳动。然而，若将剧本上传至外部 AI 处理，又面临速度慢、幻觉、信息不准确，以及剧本内容泄露风险。

因此，我决定打造一款【纯本地、零上传、专攻信息整合与查找】的剧本管理工具。

## 核心问题

长剧项目的两个固有特性，叠加出一个核心矛盾：

### 问题 1：查找与回溯极度低效

长剧集数多（30–50 集）、版本迭代频繁，项目周期又以年计。结果是：
- **文档爆炸**：同一项目可能同时存在数十个文档，找一场戏、确认一个版本，需要反复打开多个文件
- **记忆衰减**：改二稿或写续集时，前几个月的细节、伏笔往往记忆模糊，面对海量文档，回溯费时且困难

### 问题 2：多人协作下标准难以统一

不同集数常由不同编剧负责，统稿、校对和合场景时，场头表述不一致（如有人写"卧房"、有人写"卧室"），人工逐集核对耗时耗力。

## 解决方案

本工具将多集、多版本剧本集中管理，自动提取结构化信息，让查找、核对、回溯变得快速而准确。

| 核心问题 | 使用场景 | 对应功能 |
|---------|---------|---------|
| 文档分散，找一场戏/一句台词要开十几个文件 | 确认版本、查找台词、核对细节 | **全文搜索** — 跨集、跨版本统一检索，秒级定位 |
| 场头表述不一致，人工核对工作量大 | 统稿校对、会议同步、场景梳理 | **场次看板** — 自动提取所有场头，生成场景层级脑图，全局核对一目了然 |
| 周期长记忆模糊，前几个月的伏笔、细节想不起来 | 改二稿、写续集、把握角色语言风格 | **角色台词 + 素材库** — 一键汇总角色戏份，随手收藏关键信息，随时复用 |

## 功能一览

### 🔍 全文搜索
- 将项目内所有集数、所有版本的剧本内容合并索引
- 支持关键词秒级检索，快速定位到具体集、具体场、具体台词
- 避免在多版本文档之间反复切换

### 📋 场次看板
- 自动解析并提取每一场的场头信息（地点、时间等）
- 支持「表格视图」与「场景思维导图」两种模式
- 可导出场次表（Excel）与场景导图（PDF），便于统稿与会议同步

### 🎭 角色台词
- 按角色筛选，集中展示该角色在全剧中的所有台词与出场场次
- 便于把握角色语言风格、检查人设一致性，或为续集找感觉

### 🏷️ 素材库
- 在原文中选中任意文本，即可添加至自定义标签
- 常用于收藏：关键道具、闪回信息、好词好句、重要伏笔等
- 支持「速览模式」快速浏览素材，并可一键跳转回原文出处

## 🚀 快速开始

### 环境准备

1. 下载本仓库中的两个文件，放置于**同一文件夹**：
   - `index.html`
   - `mammoth.min.js`

2. 双击 `index.html`，在浏览器中打开（推荐 Chrome / Edge）

3. 点击「新建项目」→「导入」，选择 `.docx` 格式的剧本文件
   - 支持一次性多选导入
   - 同一集上传多个版本时，系统会自动保留并默认选择最新版

> ⚠️ 若未检测到 `mammoth.min.js`，将无法解析 `.docx` 文件。请确保两文件在同一目录下。

### 数据管理

- **备份数据**：点击左侧「备份数据」，可将当前所有项目导出为 JSON 文件
- **恢复数据**：在新设备或新浏览器中点击「恢复数据」，选择此前导出的 JSON 文件即可

## 🔒 隐私与数据安全

- **纯前端运行**：所有逻辑在浏览器本地执行，无需服务器
- **零上传、零联网**：剧本内容不会上传至任何云端
- **本地存储**：数据保存在浏览器 `localStorage` 中，清空浏览器数据会导致丢失，建议定期备份

## 🛠 技术栈

- [Vue 3](https://vuejs.org/) — 响应式 UI 框架
- [Tailwind CSS](https://tailwindcss.com/) — 原子化样式
- [Mammoth.js](https://github.com/mwilliamson/mammoth.js) — 浏览器端 `.docx` 解析
- 纯静态页面，无需 Node.js、无需构建、无需后端
