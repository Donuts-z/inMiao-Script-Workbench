# in喵剧本管理工作台

> 一个为**长剧编剧**量身打造的纯本地剧本管理工具。零后端、零上传、零代码基础也能用。

## 为什么做这个项目

长剧项目体量大、版本多、信息杂：
- 几十集剧本，每集一个 docx，找一场戏要开十几个文档
- 改到第五稿时，第一稿写的什么已经完全失忆
- 需要核对场头、查找角色台词、积累素材……全是大工程

市面上的剧本工具，要么面向**创作&格式**，要么面向**统筹制片**，对编剧日常"查、找、管"的需求都不够实用。

一气之下，纯文科生零基础，靠 AI 手搓了这个工具。

## ✨ 功能一览

| 模块 | 解决的问题 |
|------|-----------|
| 🔍 **全文搜索** | 所有集数的剧本贴在一起查，找一场戏/一句台词秒出结果 |
| 📋 **场次看板** | 自动提取场头，生成场景思维导图，核对场景/捡现成超方便 |
| 🎭 **角色台词** | 一键查看某角色的所有台词和戏份，写续集找感觉神器 |
| 🏷️ **素材库** | 好词好句、道具、闪回……选中直接丢进标签，秒级复用 |

## 🚀 快速开始

1. 下载本仓库的两个文件，放在**同一文件夹**内：
   - `index.html`
   - `mammoth.min.js`
2. 双击 `index.html`，在浏览器中打开（推荐 Chrome）
3. 点击「新建项目」→「导入」，选择你的 `.docx` 剧本文件
4. 开始使用！

> 💡 支持一次性多选导入。同一集上传多个版本时，系统会自动保留并默认选择最新版。

## 🔒 隐私说明

**纯本地运行，不上传、不联网、不保存任何数据到服务器。**

所有数据均存储在浏览器的 `localStorage` 中，换设备时可通过「备份数据」导出 JSON 文件，到新设备「恢复数据」即可。

## 🛠 技术栈

- [Vue 3](https://vuejs.org/) — 响应式前端框架
- [Tailwind CSS](https://tailwindcss.com/) — 原子化 CSS
- [Mammoth.js](https://github.com/mwilliamson/mammoth.js) — 浏览器端解析 `.docx`
- 纯前端实现，无需 Node.js / 无需后端 / 无需安装
